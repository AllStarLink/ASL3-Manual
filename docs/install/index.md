# ASL3 Installation

Welcome to the Installation section of the manual! We know you are excited to get on with trying out the new ASL3.

In this section of the manual, you will find instructions on how to create an ASL3 Appliance Raspberry Pi Image or installing ASL3 on a computer of virtual environment running Debian Linux.

Before you get started with installing the ASL3 software there are a few prerequisites.

<!-- start:prerequisites -->

- Do you have an AllStarLink node number assigned? If not, please refer to the [Getting Started](../basics/index.md) section to find out how to request a node number to connect to the AllStarLink network.

- Will your node be connected to one (or more) radios or RF modules (e.g. SA818)?

  - What TX frequencies will be used?
  - What RX frequencies will be used?
  - Will you be using CTCSS (or DCS) tone?  If so, what tones?

  We recommend programming your radio(s) prior to powering up and configuring your AllStarLink node.  This will ensure that any transmissions are on the correct frequencies right from the start.

  Note: when selecting TX/RX frequencies, be sure to use only those allowed by your amateur radio license and are appropriate for your location (i.e. consult the local band plans).

- Do you have the configuration information (initial settings) for any USB audio interfaces that will be associated with your node?

  - Does your interface provide a COR/COS carrier detection signal? If so, it is active high or low?
  - Does your interface provide a separate/external CTCSS signal?
  - Does the PTT signal need to be inverted?
  - Does your interface have a "clip" LED?
 
    This information should be available from the audo interface vendor/supplier.

<!-- stop:prerequisites -->
